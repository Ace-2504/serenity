# Phase 1 Handoff

## Objective

Phase 1 is complete when design and engineering can start execution without reopening core product assumptions.

## Phase 1 Decisions Locked

1. India-first B2C storefront with first-party inventory only.
2. Responsive web only for MVP.
3. English-only launch.
4. Login required before checkout.
5. Payments supported: UPI, cards, and Cash on Delivery.
6. Manual fulfillment at launch.
7. GST invoice support required.
8. Bulk ordering supports both quote requests and direct checkout path.
9. No supplier marketplace, no content hub, and no logistics aggregator in MVP.

## Inputs Ready For Phase 2 UX Work

1. Visual direction and brand system.
2. Sitemap and page inventory.
3. Component inventory.
4. Core customer journeys.
5. Business rules affecting checkout, support, and operations.

## Inputs Ready For Phase 4 Technical Work

1. Product model fields.
2. Order lifecycle states.
3. Admin roles and permissions baseline.
4. Event instrumentation map.
5. Bulk-order behavior expectations.

## Remaining Open Questions

1. COD maximum order value and restrictions.
2. Return and refund policy.
3. Pincode serviceability logic.
4. Public visibility of bulk discount pricing.
5. Exact font choice within approved type directions.

## Exit Criteria

1. Product, design, and engineering can all use the same taxonomy and page inventory.
2. No MVP stakeholder confuses this platform with a marketplace or C2C system.
3. Checkout, admin, and analytics teams have enough detail to scope implementation.
4. Open questions are small enough to resolve without rewriting the foundation.
