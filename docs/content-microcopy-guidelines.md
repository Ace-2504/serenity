# Content And Microcopy Guidelines

## Purpose

This document defines the copy tone and the key microcopy patterns for the phase 2 storefront experience.

## Voice Principles

1. Clear before clever.
2. Premium but not luxurious or distant.
3. Helpful and calm during errors.
4. Confident in commerce details such as price, delivery, and invoice availability.

## Writing Rules

1. Keep buttons action-led and short.
2. Keep support copy practical, not promotional.
3. Explain rules such as COD eligibility or coupon failure directly.
4. Avoid marketplace-style clutter copy such as excessive urgency banners.

## CTA Patterns

1. Primary purchase CTA: Add to cart.
2. Secondary purchase CTA: Buy now.
3. Bulk CTA: Request bulk pricing.
4. Discovery CTA: Shop notebooks, Explore pens, View all essentials.
5. Support CTA: Get help, Chat on WhatsApp, Contact support.

## Trust Copy Patterns

1. Delivery promise: Standard delivery in 3-7 days.
2. Invoice trust: GST invoice available.
3. Payment trust: Secure checkout with UPI, cards, and COD.
4. Support trust: Need help with your order? Support is one tap away.

## Error Copy Patterns

1. Search zero results: No exact matches found. Try a broader search or explore these categories.
2. Coupon invalid: This code could not be applied to your cart.
3. Coupon ineligible: This code is not valid for the current items or order value.
4. Payment failure: Payment did not go through. Your cart and details are still saved.
5. Stock failure: This item is no longer available in the selected quantity.
6. Address error: Please review the highlighted address fields.

## Empty-State Copy Patterns

1. Empty wishlist: Save products here to revisit them later.
2. Empty cart: Your cart is empty. Start with our most-loved picks.
3. No orders yet: Your orders will appear here once you place your first purchase.

## Section Copy Guidance

1. Home hero copy should lead with use case or season, not generic brand slogans.
2. Listing pages should use short helper descriptions only when they clarify category intent.
3. Product detail pages should lead with practical utility before aspirational language.
4. Checkout copy should reduce doubt and repetition.

## Open Copy Dependencies

1. Return policy language depends on final policy decision.
2. COD explanation copy depends on restriction rules.
3. Bulk pricing messaging depends on public visibility decision.
